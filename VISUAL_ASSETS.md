# Visual Assets & Brand Guidelines

## Color Palette

| Color | Hex | Usage |
|-------|-----|-------|
| Bitcoin Orange (Primary Accent) | `#F7931A` | Buttons, highlights, key numbers |
| Dark Background | `#0A0E17` | Page backgrounds, dark mode |
| Card Background | `#111827` | Content cards, panels |
| Light Text | `#E2E8F0` | Primary body text |
| Secondary Text | `#94A3B8` | Captions, labels |
| Positive/Green | `#22C55E` | Balances, success states |
| Alert/Red | `#EF4444` | Errors, warnings |
| Info/Blue | `#3B82F6` | Links, secondary actions |

## Typography

| Usage | Font | Fallbacks |
|-------|------|-----------|
| Code / Addresses | SF Mono | Fira Code, Cascadia Code, monospace |
| Headings | System UI | -apple-system, Segoe UI, sans-serif |
| Body | System UI | Same as headings |

BTCPC uses monospace fonts extensively — it reinforces the technical, engineering-first identity.

## Logo

No official logo has been finalized. When creating one:

**Do:**
- Reference Bitcoin's orange (we're building on the idea)
- Include a computation/AI element (neurons, circuits, thought bubbles)
- Keep it simple enough to work at 16x16 favicon size
- Consider the number 42 as a subtle element

**Don't:**
- Copy Bitcoin's logo
- Use generic blockchain imagery (chains, cubes)
- Make it too complex for small sizes

## The Number 42

42 appears throughout BTCPC as a design signature:
- 42,000,000 total supply
- Port 4242 for the block explorer (42 x 101)
- 42 BTCPC fee to create a custom token
- 4,200 character limit for block inscriptions (42 x 100)

Use this in marketing — it's a recognizable cultural reference that makes the brand memorable and approachable.

## Screenshots & UI

The block explorer (btcpcscan) at `localhost:4242` is the primary visual reference:
- Dark theme with Bitcoin orange accents
- Stats cards showing chain metrics
- Block detail pages with hash chains, state roots, ledger entries
- Account pages with balances, transaction history, state proofs
- Settings page with CPU/GPU sliders

## Tone in Visuals

- **Technical but inviting** — show real data, real blocks, real transactions
- **Not corporate** — avoid stock photos, generic tech imagery
- **Engineering-authentic** — terminal screenshots, code snippets, network diagrams are fine
- **The "dream" metaphor** — use it in copy and visuals. The chain dreams. Tokens are dreams. Mining is dreaming.
